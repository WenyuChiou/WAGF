# reproducibility_manifest.json schema (canonical)

Written by `broker/core/experiment_runner.py:_collect_reproducibility_metadata`
at run completion. Located at `<run_dir>/reproducibility_manifest.json`.

## Required fields (REQUIRE for GREEN verdict)

| Field | Type | Meaning | Why required |
|-------|------|---------|--------------|
| `model` | str | Ollama model tag (e.g. `gemma3:4b`, `gemma4:e2b`) | Identifies LLM that produced the trace |
| `seed` | int | Random seed | Reproducibility |
| `git_commit` | str (40-char hex) | HEAD at run-start | Maps data to code state |
| `git_dirty` | bool | uncommitted changes at run-start | If true, run cannot be reproduced |
| `governance_profile` | str | `strict` / `disabled` / etc. | Identifies experiment arm |
| `agent_types_config` | path | YAML file consumed | Maps to declarative config |
| `config_hash` | str (sha256) | hash of config + bat command | Detects silent config drift |
| `timestamp` | ISO 8601 | run start time | Cross-check against trace mtimes |

## Strongly-recommended fields

| Field | Type | Why |
|-------|------|-----|
| `temperature` | float | LLM sampling parameter (paper claim) |
| `top_p` | float | LLM sampling parameter (paper claim) |
| `num_ctx` | int | Context window |
| `num_predict` | int | Max output tokens |
| `thinking_mode` | str | `disabled` / `enabled` (Ollama feature) |
| `model_digest` | str (sha256) | Ollama digest, catches silent model upgrades |
| `model_family` | str | e.g., `gemma4` |
| `model_parameter_size` | str | e.g., `5.1B` |
| `model_quantization` | str | e.g., `Q4_K_M` |
| `python_version` | str | for env reproducibility |

## Optional

| Field | Type | Why |
|-------|------|-----|
| `num_years` | int | sim horizon |
| `prompt_flags` | list[str] | enabled CLI flags |
| `runner_script` | str | path to entry-point bat |
| `notes` | str | free-text |

## Validation rules

1. `git_dirty == true` → automatic RED with reason "uncommitted changes
   at run-start; cannot be reproduced".
2. `git_commit` not in repo's git log → RED "manifest references unknown
   commit".
3. `agent_types_config` file does not exist or differs from
   `git_show <git_commit>:<agent_types_config>` → RED "config drift".
4. `model_digest` mismatch vs current `ollama list` digest → YELLOW
   "model may have been re-pulled since run".
5. Missing any required field → RED.
6. Missing any strongly-recommended field → YELLOW with severity =
   medium.

## Cross-reference: data trace timestamp vs commit date

The most failure-prone check. For each run:

```bash
# 1. read first trace timestamp
first_ts=$(head -1 <run_dir>/raw/*.jsonl | python -c "import json,sys; print(json.loads(sys.stdin.read())['timestamp'])")

# 2. read commit date for the relevant code (e.g., irrigation_env.py)
commit_date=$(git show -s --format=%cI <git_commit> -- examples/irrigation_abm/irrigation_env.py)

# 3. compare
# trace_ts >= commit_date → POST_FIX_OK
# trace_ts <  commit_date → INVALID_BASELINE (the data was generated by
#   pre-commit code, even though the dir name suggests post-commit)
```

This single check catches both the 2026-04-19 priority-schema confound
(via `prompt_flags` field absence/presence) and the 2026-04-25 v21
dir-naming mismatch (via `git_commit` field cross-ref against
`irrigation_env.py:588` v21 fix commit `4be5092`).

## What is NOT in the manifest (and why this matters)

The manifest does not currently capture:

- `window_size` (memory window) — depends on `agent_types_config`
- `memory_engine_type` (humancentric / universal / etc.)
- `prompt_flags[]` — the priority-schema confound would have been caught
  earlier if this list were always written
- `bat_path` — exact bat script that produced the run
- `random_seed_for_environment` — distinct from `seed` (which is the LLM seed)

Pre-submission checklists (this skill's `references/checklist.md`) flag
any missing field as YELLOW.
